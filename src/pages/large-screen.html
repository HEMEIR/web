低风险漏洞

代码逻辑不严谨，但不会导致严重后果。例如：缺少输入验证、日志记录不完整等。

// 示例：缺少输入验证
function transfer(address to, uint amount) public {
    balances[msg.sender] -= amount;
    balances[to] += amount;
}

中风险漏洞

可能导致资产损失或功能异常。例如：整数溢出、权限控制不当等。

// 示例：整数溢出风险
function withdraw(uint amount) public {
    require(balances[msg.sender] >= amount);
    balances[msg.sender] -= amount;
    payable(msg.sender).transfer(amount);
}

高风险漏洞

可能导致严重资产损失或系统完全崩溃。例如：重入攻击、逻辑漏洞等。

// 示例：重入攻击风险
function withdraw(uint amount) public {
    require(balances[msg.sender] >= amount);
    payable(msg.sender).transfer(amount);
    balances[msg.sender] -= amount;
}

关键风险漏洞

直接导致合约完全失效或所有资产被盗。例如：私有变量可修改、构造函数漏洞等。

// 示例：构造函数漏洞（旧版Solidity）
function Wallet() public {
    owner = msg.sender;
}

// 被错误调用为普通函数，导致owner被篡改
function wallet() public {
    owner = msg.sender;
}
